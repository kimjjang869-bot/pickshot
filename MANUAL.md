## 🔥 PickShot은 이런 문제를 해결합니다

### "촬영 끝났는데 선별·전달이 너무 오래 걸려요"

| 현장의 고통 | PickShot이 해결하는 방식 |
|---|---|
| 촬영 후 3,000장 선별에 반나절 걸림 | 투어박스/키보드로 **초당 30장 탐색**, 별점 1~5 / 컬러라벨 즉시 분류 |
| 카드 5장 백업하느라 대기 시간 낭비 | 카드 꽂으면 **자동 감지 → 복사 → 검증 → 언마운트 → 다음 카드 자동 시작** |
| "이 중에 어떤 게 좋은지 골라 주세요" | **클라이언트 셀렉**: 업로드 즉시 링크/QR 생성 → 고객이 웹에서 직접 픽 |
| 행사 당일 납품 압박 | **RAW→JPG 배치 변환** + 픽쳐스타일 + 워터마크, GPU 가속으로 수백 장도 몇 분 |
| 라이트룸 넘기기 전 사전 선별 | **XMP 사이드카 + Lightroom 호환 컬러라벨/별점** 즉시 연동 |
| 사진이 너무 많아 카테고리별로 나누기 지침 | **AI 분류**가 카테고리별 폴더 자동 생성 + 파일 이동 |
| RAW 프리뷰가 느려서 답답 | **HW JPEG 디코더 + Metal GPU + 임베디드 썸네일 프리페치**로 즉시 표시 |

## 🎯 핵심 기능 TOP 7 — 왜 이 기능이 필요한가

### 1. 👥 클라이언트 셀렉 (Client Select) — 고객 직접 픽
> **"고객한테 메신저로 사진 수백 장 돌리지 마세요"**

촬영 후 바로 사진을 Google Drive에 업로드하고, **공유 링크와 QR 코드**를 생성해서 클라이언트에게 전달합니다. 클라이언트는 웹 브라우저에서 직접 사진을 보고 마음에 드는 걸 체크하고, 결과가 PickShot으로 돌아와서 **자동으로 해당 사진에 셀렉 마크**가 찍힙니다.

**언제 씁니까?**
- 웨딩/스냅/프로필 촬영 후 클라이언트가 "이거 이거 골라 주세요" 해야 할 때
- 기업 행사 담당자에게 사전 검수를 맡겨야 할 때
- 여러 담당자가 각자 원하는 컷을 골라야 할 때

**사용법:**
1. 툴바 "클라이언트" 버튼 → "사진 업로드 + 링크 생성"
2. 업로드 완료 후 링크/QR 자동 표시
3. 클라이언트가 웹에서 픽 → 셀렉 파일(.pickshot) 받아서 "가져오기"
4. Drive 업로드 시 자동으로 받아올 수도 있음

---

### 2. ☁️ G 셀렉 (G Select) — 실시간 Google Drive 업로드
> **"현장에서 찍자마자 바로 공유해야 할 때"**

`G` 키 한 번 누르면 현재 사진이 Google Drive로 즉시 업로드됩니다. 세션 링크가 만들어져 있어서 클라이언트/편집자/디렉터가 **촬영 중에도 실시간으로** 업로드된 사진을 볼 수 있습니다.

**언제 씁니까?**
- 광고/브랜드 촬영에서 디렉터가 현장 밖에서 모니터링할 때
- 여러 명이 협업하는 팀 촬영
- "지금 찍은 거 어떻게 나왔어?" 즉답이 필요할 때

**사용법:**
1. 툴바 "G" 버튼 → 세션 시작
2. 사진 선택 후 `G` 키 → 즉시 업로드
3. 공유 링크/QR을 팀에 전달 → 끝

### 3. 💾 메모리카드 자동 백업 (연속 백업)
> **"카드 5장을 안 지키고 자리 비울 수 있어야 합니다"**

SD/CF 카드를 꽂기만 하면 자동으로 DCIM 폴더를 감지하고 백업을 시작합니다. 각 파일은 크기 비교로 **검증**되고, 실패 시 **3회 재시도**, 성공 시 카드가 **자동 언마운트**됩니다. 그리고 다음 카드를 꽂으면 **같은 폴더로 알아서 이어서 백업**합니다.

**언제 씁니까?**
- 행사 촬영 종료 직후, 카드 여러 장을 한꺼번에 옮길 때
- 잠시 자리 비우는 동안 백업 돌리고 싶을 때
- "카드 잃어버리면 끝장" 상황에서 검증까지 자동화하고 싶을 때

**작동 순서:**
```
카드1 꽂기 → 자동 감지 → 복사 → 검증 → 언마운트 → "다음 카드 대기"
카드2 꽂기 → 자동으로 같은 폴더로 백업 시작 → (반복)
```

### 4. 🤖 AI 분류 + 자동 폴더 정리
> **"사진 3,000장을 인물/풍경/디테일/비하인드로 나누기"**

Claude Haiku/Sonnet 또는 Gemini Flash/Pro로 사진을 분석해서 카테고리별로 분류하고, **실시간으로 해당 카테고리 폴더를 만들어서 파일을 이동**시킵니다. 프리셋 분류도 있지만 **커스텀 프롬프트**로 "이 촬영은 의상별로 나눠줘" 같은 자유 분류도 가능합니다.

**언제 씁니까?**
- 대량 촬영 후 한꺼번에 정리가 필요할 때
- 특정 기준(의상, 장소, 피사체)으로 분류해야 할 때
- API 돈 쓰기 싫으면 **Vision 로컬 분류(무료)** — 22개 카테고리 자동 분류

**지원 엔진 & 비용:**
| 엔진 | 특징 | 1000장 기준 |
|---|---|---|
| Vision 로컬 | 인터넷 불필요, 완전 무료 | $0 |
| Gemini Flash | 저렴, 빠름 | ~$0.15 |
| Claude Haiku | 균형 | ~$0.50 |
| Gemini Pro | 고정확도 | ~$2.50 |
| Claude Sonnet | 최고 정확도 | ~$6.00 |

### 5. ⚡ 초고속 RAW 뷰잉
> **"RAW 프리뷰가 JPG만큼 빨라야 합니다"**

일반 뷰어에서는 RAW 파일을 클릭할 때마다 5~10초 디코딩을 기다려야 합니다. PickShot은 **VideoToolbox 하드웨어 JPEG 디코더 + Metal GPU + 임베디드 썸네일 프리페치**를 활용해서, RAW 파일의 내장 프리뷰를 즉시 표시합니다.

**내부 최적화:**
- IOSurface 기반 GPU 메모리 공유 (CPU↔GPU 복사 없음)
- 양방향 30장 프리페치 (앞뒤로 미리 로딩)
- NSCache L1 + 디스크 캐시 2GB
- 투어박스/키보드 네비게이션 **0ms 캐시 히트**

**결과:** 2,000장+ 폴더도 끊김 없이 스크롤/넘김 가능

### 6. 🎨 Lightroom 완벽 호환 (XMP + 컬러라벨)
> **"PickShot에서 고른 선별을 Lightroom에서 그대로 보기"**

별점(1~5), 컬러 라벨(빨강/노랑/초록/파랑/보라)을 매기면 **XMP 사이드카 파일**에 자동 저장됩니다. Lightroom에서 해당 폴더를 가져오면 별점/라벨이 그대로 표시됩니다. 반대로 Lightroom에서 평가한 파일도 PickShot에서 열면 표시됩니다.

**v8.0에서 달라진 점:**
- SP 셀렉 → **별점 5**로 전환 (스페이스바 = 별점 5 토글)
- 컬러 라벨 5종 (Lightroom 표준: Red/Yellow/Green/Blue/Purple)
- XMP `xmp:Label` 필드에 한글명이 아닌 **영문 표준명**으로 저장 → Lightroom 호환 100%

### 7. 📦 RAW→JPG 배치 변환 + 워터마크
> **"당일 납품, 해상도/화질/워터마크 한 번에"**

RAW 파일을 일괄로 JPG로 변환하면서 리사이즈, 샤프닝, 색공간 변환, **텍스트/이미지 워터마크**를 동시에 적용합니다. GPU 가속으로 수백 장도 몇 분 만에 처리됩니다.

**사용 옵션:**
- 해상도: 원본 / 4000px / 3000px / 2000px
- 품질: 최고 / 높음 / 보통 / 낮음
- 색공간: sRGB / Display P3 / Adobe RGB
- 샤프닝: 없음 / 약간 / 보통 / 강하게
- 수평 보정: Vision 자동 수평 감지
- 워터마크: 텍스트 또는 PNG 로고, 위치/투명도/크기 조절
- 파일명 패턴: 원본명 / 접두사+번호 / 날짜 포함

## 1. 시작하기

### 1.1 설치
1. DMG 파일 다운로드 → 열기
2. PickShot 아이콘을 **Applications** 폴더로 드래그
3. Applications에서 PickShot 실행
4. 첫 실행 시 Gatekeeper 경고가 나오면 시스템 설정 > 개인정보 보호 및 보안에서 "확인 없이 열기"

### 1.2 첫 실행
시작 화면에서:
- **폴더 열기** 또는 `Cmd + O`
- 또는 폴더를 앱 창에 **드래그 앤 드롭**

> JPG 폴더와 RAW 폴더가 분리되어 있어도 **자동 매칭**됩니다.

## 2. 화면 구성

```
+-------------------------------------------------------+
| 툴바 Row 1: 홈 · 경로 · 클라이언트 · 내보내기 · 매칭 · AI |
+-------------------------------------------------------+
|         | 툴바 Row 2: 별점 · 라벨 · 검색 · 정렬 · AI  |
|  폴더    +------------------+----------------------+-+
|  트리    |                  |                      | |
|          |  썸네일 그리드    |      미리보기         | |
|          |                  |                      | |
|          +------------------+----------------------+ |
|          | 상태바: 선택/전체/폴더용량                 |
+----------+------------------------------------------+
```

### 툴바 Row 1 — 액션
홈 · 폴더 브라우저 토글 · 경로 · 클라이언트 셀렉 · 내보내기 · 매칭 · AI 셀렉 · 인물 그룹 · G 셀렉

### 툴바 Row 2 — 필터 (v8.0 신규 레이아웃)
- **왼쪽**: 별점 All/1/2/3/4/5 · 컬러 라벨 5종 · 검색
- **중앙** (썸네일 칸 너비가 충분할 때): 정렬 · AI 필터 · 스마트 컬렉션
- **오른쪽**: 레이아웃 전환 · 전체화면 · 듀얼뷰 · 비교 · 표정비교 · 더보기

> v8.0의 Row 2는 **썸네일 패널 너비에 연동**됩니다. 드래그 핸들로 썸네일 칸을 좁히면 중앙 그룹이 자동으로 숨겨지고, 넓히면 다시 나타납니다.

## 3. 기본 사용법

### 3.1 보기 모드 전환
| 단축키 / 버튼 | 모드 |
|---|---|
| 툴바 레이아웃 버튼 | 그리드+미리보기 ↔ 필름스트립 |
| `Cmd + F` | 전체화면 |
| `D` | 듀얼 뷰어 |
| `C` (2~4장 선택) | 비교 보기 |
| 얼굴 아이콘 (2~6장) | 표정 비교 |

### 3.2 썸네일 탐색
| 단축키 | 동작 |
|---|---|
| `←` `→` | 이전/다음 사진 |
| `↑` `↓` | 위/아래 행 |
| `Shift + 방향키` | 범위 선택 확장 |
| `Cmd + A` | 전체 선택 |
| `Cmd + D` | 전체 해제 |
| `Cmd + 클릭` | 개별 추가/해제 |
| `Shift + 클릭` | 범위 선택 |

### 3.3 썸네일 크기
툴바 하단 상태바에서 크기 슬라이더로 조절 (60~250px).

### 3.4 미리보기 줌
| 단축키 | 동작 |
|---|---|
| `Cmd + 0` | 화면 맞춤 |
| `Cmd + =` / `-` | 확대/축소 |
| 스크롤 | 포커스 중심 확대/축소 |
| 더블클릭 | 350% ↔ 화면 맞춤 |
| 미들클릭 | 화면 맞춤 |
| `H` | 히스토그램 토글 |
| `I` | EXIF 메타데이터 토글 |

### 3.5 정렬
툴바 Row 2 중앙의 정렬 메뉴:
- 촬영시간 (오래된순/최신순)
- 파일명 (오름/내림)
- 별점 (높은순/낮은순)

### 3.6 검색
툴바 Row 2 왼쪽의 검색창에 파일명 입력 → 실시간 필터링. 결과 수가 배지로 표시.

### 3.7 스마트 컬렉션
현재 적용된 필터(별점/라벨/AI/검색 등)를 이름 붙여 저장해두고 한 번에 복원할 수 있습니다. 툴바 Row 2의 필터 아이콘 → "현재 필터 저장...".

## 4. 사진 선별

### 4.1 별점 매기기 (v8.0)
| 키 | 동작 |
|---|---|
| `1` ~ `5` | 별점 1~5점 |
| `0` | 별점 초기화 |
| `Space` | **별점 5 토글** (빠른 픽) |

여러 장 선택 시 숫자키를 누르면 **선택된 모두에 동시 적용**됩니다.

**별점 필터:** 툴바 Row 2 왼쪽의 별점 Pill (All/1/2/3/4/5) 클릭 → 해당 별점 이상 표시.

### 4.2 컬러 라벨 (v8.0 재편)
Lightroom 호환 5색 컬러 라벨:

| 키 | 색상 | 영문 (XMP) |
|---|---|---|
| `6` | 빨강 | Red |
| `7` | 노랑 | Yellow |
| `8` | 초록 | Green |
| `9` | 파랑 | Blue |
| 우클릭 메뉴 | 보라 | Purple |

**라벨 필터:** 툴바 Row 2의 컬러 라벨 원 클릭 → **다중 선택 가능** (빨강+초록 동시 필터 등).

**Lightroom 호환:** XMP 사이드카에 영문 표준명으로 저장되어 Lightroom 표시 이슈 없음.

### 4.3 비교 보기
2~4장 선택 후 `C` 키 → 나란히 비교.

### 4.4 표정 비교
2~6장 선택 후 툴바의 얼굴 아이콘 → 각 사진의 얼굴만 확대해서 비교. 인물 사진 선별에 유용.

### 4.5 Quick Look / 실행 취소
- `P` — macOS Quick Look
- `Cmd + Z` — 최근 별점/라벨 변경 되돌리기

## 5. 클라이언트 셀렉 & G 셀렉

### 5.1 클라이언트 셀렉 (Client Select)

**사용 순서:**
1. 사진 로딩 후 툴바 "클라이언트" 메뉴 → "사진 업로드 + 링크 생성"
2. Google 계정 로그인 (처음 한 번)
3. 업로드 대상 폴더 선택 (예: "2026-04 김민수 결혼식")
4. 업로드 진행 → 완료 시 **공유 링크 + QR** 표시
5. 클라이언트에게 링크/QR 전달
6. 클라이언트가 웹에서 픽 완료 후 **셀렉 파일(.pickshot)**을 받아옴
7. PickShot 툴바 "클라이언트" → "셀렉 파일 가져오기" 또는 "Drive에서 가져오기"
8. 셀렉된 사진에 자동으로 **별점 5 + 빨강 라벨** 적용

### 5.2 G 셀렉 (G Select)

**현장에서 실시간 공유:**
1. 툴바 "G" 버튼 → 세션 시작
2. 세션 링크/QR을 디렉터/팀에 전달
3. 사진 선택 후 `G` 키 → 즉시 업로드
4. 팀원이 웹/앱에서 실시간으로 확인

G 셀렉 상태는 썸네일에 **G 배지**로 표시됩니다.

## 6. 메모리카드 백업

### 6.1 활성화
설정 (`Cmd + ,`) → 일반 → **"메모리카드 자동 백업"** 활성화.

### 6.2 사용법
1. SD/CF 카드를 맥에 연결
2. 백업 프롬프트 자동 표시 → 대상 폴더 선택
3. 백업 자동 시작 (진행률 + 속도 + ETA 표시)
4. 각 파일: **크기 검증 → 실패 시 3회 재시도**
5. 완료 시 카드 **자동 언마운트** ("추출해도 안전합니다" 알림)
6. "다음 카드 대기" 모드 진입
7. **새 카드 연결 → 같은 폴더로 자동 백업**

### 6.3 멀티 세션
카드를 여러 장 동시 연결하면 각각 별도 세션으로 병렬 백업 진행.

### 6.4 중복 처리
이미 대상 폴더에 존재하는 파일은 **크기 동일 시 자동 건너뛰기**. 크기가 다르면 사용자 선택(덮어쓰기/이름 변경/건너뛰기).

## 14. FAQ

### Q. 클라이언트 셀렉과 G 셀렉의 차이는?
**A.** 클라이언트 셀렉은 **일회성 배치 업로드 + 웹에서 픽**, G 셀렉은 **실시간 스트리밍 업로드**입니다. 납품용이면 클라이언트 셀렉, 현장 모니터링용이면 G 셀렉이 맞습니다.

### Q. SP 셀렉은 없어졌나요?
**A.** v8.0에서 SP 셀렉은 **별점 5**로 통합됐습니다. 스페이스바 = 별점 5 토글로 같은 기능입니다. 기존 SP 데이터는 별점 5로 자동 마이그레이션됩니다.

### Q. 컬러 라벨이 Lightroom에서 보이지 않아요
**A.** v8.0부터 XMP에 영문 표준명(Red/Yellow/Green/Blue/Purple)으로 저장됩니다. Lightroom의 컬러 라벨 세트가 기본값(Lightroom Default)인지 확인하세요. 기존에 한글로 저장된 사진은 다시 라벨을 찍으면 자동 교정됩니다.

### Q. RAW 파일이 표시되지 않아요
**A.** PickShot은 JPG 기준으로 미리보기하고 RAW를 자동 매칭합니다. RAW만 있는 경우에도 내장 프리뷰를 사용하지만, 최신 RAW 포맷(예: Nikon Z8/Z9 High Efficiency)은 디코딩이 오래 걸릴 수 있습니다.

### Q. 사진 로딩이 느려요
**A.** 설정 > 성능 최적화에서 벤치마크 → "속도 우선" 또는 "균형" 프로필 적용. 미리보기 해상도를 2000px 이하로 낮추세요.

### Q. AI 분류가 안 돼요
**A.** 설정 > AI 엔진에서 API 키 입력 + "테스트" 버튼으로 확인. **Vision 로컬 분류는 API 키 없이 무료**로 사용 가능합니다.

### Q. 메모리카드 백업 프롬프트가 안 떠요
**A.** 설정 > 일반 > "메모리카드 자동 백업"이 켜져 있는지 확인. DCIM 폴더가 있는 카드만 감지됩니다.

### Q. 캐시가 너무 큰데 줄이는 방법?
**A.** 설정 > 캐시 → 현재 크기 확인 → "썸네일/미리보기/전체 캐시 삭제". 썸네일 캐시 최대값을 1~2GB로 제한하세요.

### Q. 우클릭 메뉴로 뭘 할 수 있나요?
**A.** 별점/라벨 매기기 · SP/G 셀렉 · 내보내기 · RAW→JPG · Finder로 복사 (최근 폴더 목록) · 파일명 복사 · Finder에서 보기 · **연결 프로그램으로 열기** (Photoshop/Lightroom/Bridge/Capture One 자동 감지) · 이름 변경 · 목록에서 제거 · 새 폴더.

### Q. 투어박스 연동이 되나요?
**A.** 투어박스 SDK는 기본 지원이며, 캐시 히트율 0ms에 최적화돼 있어 30장/초 탐색도 끊김 없습니다. 키 매핑은 시스템 키보드 매핑과 동일하게 동작합니다.

### Q. 전체 삭제가 위험한가요?
**A.** PickShot의 삭제는 **휴지통 이동**이므로 복구 가능합니다. Backspace 키로 원본 삭제 설정은 기본 꺼짐이며, 필요 시 설정에서 활성화할 수 있습니다.
