PickShot 매뉴얼

현장에서 30분에 끝내는
사용법.

10,000장을 추리는 가장 빠른 길. 설치부터 클라이언트 전달까지 30분 안에 몸에 붙이는 순서대로 정리했습니다.

버전 v8.9 기준 · 최종 업데이트 2026-04-22 · 읽는 시간 약 12분
01

빠른 시작 — 5분 요약

자세한 설명이 필요 없다면 아래 5단계만 따라 해도 현장에서 바로 씁니다.

  1. 1
    폴더 열기
    ⌘O 또는 Finder에서 사진 폴더를 드래그 앤 드롭. 2,000장 RAW면 약 1.5초에 떴습니다.
  2. 2
    별점으로 훑기
    방향키로 이동, 숫자키 1~4로 등급. 베스트컷은 Space로 별 5점 원핸드 셀렉.
  3. 3
    클라이언트용 픽은 G
    별점과 독립된 레이어. 클라이언트에게 보낼 컷만 G로 묶습니다.
  4. 4
    필터 + 내보내기
    상단 필터에서 ★5 또는 G 선택 → ⌘E로 폴더 지정 내보내기.
  5. 5
    클라이언트에게 링크 전달
    Google Drive 한 번 연결 후 공유 → 링크 복사. 별점/펜/코멘트가 내 Mac에 실시간 반영됩니다.
현장 팁

촬영 중이라면 2번 단계부터 시작하세요. 메모리카드를 꽂으면 백업이 자동으로 돌면서 PickShot에 실시간으로 썸네일이 채워집니다. 복사가 끝날 때쯤이면 셀렉도 같이 끝나 있습니다.

설치 순서

  1. GitHub 릴리즈에서 최신 PickShot_v8.9.x.dmg 다운로드
  2. DMG를 열어 PickShot을 응용 프로그램 폴더로 드래그
  3. 처음 실행 시 Gatekeeper가 차단하면 — 시스템 설정 → 개인정보 및 보안 → "확인 없이 열기" 클릭 (Apple Developer ID 공증 완료됐지만 신규 앱은 처음 한 번만 필요)
  4. 권한 요청이 뜨면 파일 및 폴더 접근 · 이동식 볼륨 모두 허용
  5. Google Drive 연동을 쓰려면 설정 → 통합 → Google 로그인. OAuth 2.0 토큰은 Keychain에만 저장됩니다.
주의

외장 SSD에서 직접 작업할 수 있지만, USB 3.2 Gen 2 (10 Gbps) 이상을 권장합니다. USB-C 허브에 물린 구형 SSD에서는 캐시가 병목이 됩니다.

03

현장 워크플로우 — 10,000장을 30분에

제가 매주 페스티벌에서 쓰는 순서 그대로 옮겨 놓았습니다. 상황에 맞게 일부만 쓰셔도 됩니다.

0분
메모리카드 삽입

카드 리더를 꽂는 순간 PickShot이 감지합니다. 복사 대상 폴더는 년월일_행사명 형식으로 자동 생성 (수정 가능). 복사가 시작되면서 동시에 썸네일이 채워집니다.

0~3분
복사 + 1차 훑기 동시 진행

채워지는 썸네일을 방향키로 훑으면서 명백한 실패컷 X로 제외, 베스트 후보 Space로 별 5점. 복사가 끝나기 전에 약 30% 선셀렉이 됩니다.

3~15분
본 셀렉

필터를 ★3 이상으로 걸고 집중 훑기. 비슷한 컷은 연사 그룹으로 자동 묶여 베스트만 보입니다 (v8.9 CLIP 유사도). 얼굴 그룹 필터로 특정 인물만 볼 수도 있습니다.

15~22분
클라이언트용 G 픽

별 5점 중 클라이언트에게 바로 보낼 컷에만 G. 필요하면 얼굴 태그로 "박신혜 중심 12컷"처럼 따로 묶기.

22~27분
비파괴 보정 + 내보내기

G 픽에만 간단한 보정 프리셋 일괄 적용 (화이트밸런스·노출·컬러). ⌘E로 JPG 또는 DNG 내보내기. XMP는 Lightroom에서 그대로 읽힙니다.

27~30분
클라이언트 전달

내보내기 중 Google Drive 자동 업로드. 공유 링크 복사 후 카톡으로 전달. 클라이언트가 브라우저에서 별점/코멘트 남기면 내 PickShot에 실시간 반영됩니다.

왜 이렇게 빠른가

Lightroom은 카탈로그를 만든 뒤 셀렉을 시작합니다. PickShot은 파일이 도착하는 대로 바로 셀렉합니다. 대기 시간 자체가 0에 수렴하기 때문에 체감 속도가 다릅니다.

04

G-Select — 내 기준과 클라이언트용을 분리

가장 자주 받는 질문: "별점이 있는데 G가 왜 또 필요하나요?" 답은 간단합니다 — 평가 기준이 다르기 때문입니다.

별점 1~5
내 기준 — 사진의 절대 품질
  • 초점·노출·구도·표정 기준의 품질 등급
  • 한 번 정하면 잘 안 바꾸는 레이어
  • 포트폴리오·아카이브·연간 베스트 고를 때 기준
  • 5점은 Space로 원핸드 빠른 셀렉
G 켜짐 / 꺼짐
클라이언트용 — 이번 전달에 넣을 컷
  • 클라이언트 요구·컨셉에 맞는 선별
  • 프로젝트마다 갈아끼우는 레이어
  • 별점 3점인데 클라이언트가 좋아할 순간 → G
  • 별점 5점인데 분위기 안 맞음 → G 제외

실전 예시

페스티벌 촬영 2,000장
★5 80장 기술·감정 모두 완성된 내 베스트
G 120장 이번 클라이언트(음반사)가 요구한 "아티스트 중심, 관중 리액션"에 맞는 컷
★5 ∩ G 45장 둘 다 해당 — 메인 전달용
★5 − G 35장 내 베스트지만 이 클라이언트엔 안 맞음 — 포트폴리오 보관

06

메모리카드 자동 백업

이 기능 하나 때문에 PickShot을 쓰기 시작한 분이 많습니다. 한 번 설정하면 현장에서 카드만 꽂으면 끝입니다.

최초 1회 설정

  1. 설정 → 메모리카드 백업 열기
  2. 대상 폴더 지정 — 보통 외장 SSD 루트 (/Volumes/SSD-WORK/)
  3. 폴더 이름 규칙 선택 — YYYYMMDD_행사명이 기본. 촬영마다 행사명만 바꾸면 됩니다.
  4. 완료 후 언마운트 체크 — 복사 끝나면 카드가 안전하게 튕겨 나옵니다.
  5. 해시 검증 (SHA-256) 체크 — 원본/사본 무결성 확인. 1~2% 시간 추가되지만 안전합니다.

현장에서 쓰는 법

1
카드 삽입

PickShot이 자동 감지하고 복사 시작. 진행률은 우하단 상태바에 실시간 표시.

2
복사 중 셀렉

복사가 끝나기를 기다릴 필요 없이, 채워지는 썸네일을 바로 훑습니다.

3
자동 언마운트

완료되면 카드가 알아서 언마운트. 빼서 다음 카드를 꽂으면 같은 행사 폴더에 이어 붙습니다.

4
6장 연속 촬영 중 손 놓기

카드 6장을 연속으로 꽂아도 대기 없이 전부 백업. 촬영에만 집중할 수 있습니다.

안전 원칙

PickShot은 카드에서 삭제하지 않습니다. 백업이 끝나고 SSD에서 실제 파일 확인 후, 카드는 카메라에서 포맷하세요. 해시 검증을 켰다면 검증 완료 ✓ 표시를 확인하고 포맷하시길 권합니다.

07

적극 캐시 모드

툴바의 ⚡ 아이콘. 폴더 진입 즉시 전체 프리뷰를 병렬로 미리 캐싱합니다. 10,000장 폴더도 40초에 풀 예열.

언제 켜야 하나

  • 행사 끝나고 사무실 돌아왔을 때 — 커피 한 잔 하는 사이 전부 예열
  • 다음 날 다시 열 폴더 — 닫기 전에 캐시 유지 옵션만 켜두면 그대로 유지
  • 클라이언트 미팅 직전 — 회의 들어가기 전 1~2분으로 풀스피드 보장

언제 꺼야 하나

  • 현장에서 배터리 아끼고 싶을 때 — 팬 속도·전력 최소화
  • 디스크 공간 빠듯할 때 — 캐시 최대 용량을 설정에서 제한 가능 (기본 2 GB)
v8.9 변경점

메모리 상주량을 18 GB → 221 MB로 줄였습니다 (98.8% 감소). 적극 캐시를 켜도 램을 잡아먹지 않으니 안심하고 쓰세요.

08

클라이언트 뷰어 — 링크 한 줄로 끝

클라이언트는 로그인도, 앱 설치도 필요 없습니다. 브라우저만 있으면 별점·펜·코멘트로 의사를 전달할 수 있습니다.

최초 1회 연결

  1. 설정 → 통합 → Google 계정 연결
  2. OAuth 동의 창에서 Drive 파일 업로드 권한만 허용 (전체 접근 아님)
  3. 업로드 대상 폴더 지정 (기본: PickShot/Client)

공유 순서

  1. G 픽 또는 ★5 컷을 ⌘E로 내보내기 (웹사이즈 JPEG 자동 생성)
  2. 완료되면 툴바에 공유 링크 복사 버튼 활성화
  3. 클립보드에 복사된 링크를 카톡·메일로 전달
  4. 클라이언트의 액션(별점·코멘트)은 내 PickShot 상단 알림에 실시간 도착
프라이버시

RAW 원본은 절대 업로드되지 않습니다. 2560px 웹사이즈 JPEG만 올라가고, 링크에는 만료 기한(기본 30일)을 걸 수 있습니다.

09

추가 기능

얼굴 그룹 (AdaFace R18)

같은 사람 자동 그룹핑. 99.82% 정확도. 모든 추론이 온디바이스에서 돌아가므로 서버로 전송되는 데이터는 없습니다. 인물에 이름을 태그하면 이후 "박신혜"만 필터로 걸어서 볼 수 있습니다.

연사 베스트 자동 선별 (v8.9)

CLIP 유사도로 비슷한 컷을 그룹으로 묶고, 초점·표정·구도 점수로 베스트 1장을 추천. 연사 10장 중 9장 자동 숨김 → 결정 시간 1/10.

같은 옷 검색 (v8.9)

한 장 선택 후 "비슷한 옷". CLIP 임베딩이 같은 의상을 입은 컷을 찾아줍니다. 웨딩·패션·인터뷰에서 룩 단위 셀렉에 유용합니다.

사용자 학습 (v8.9)

셀렉한 사진들의 패턴에서 개인 취향 프로필을 자동 생성. 다음 폴더에서 AI 추천이 내 스타일에 맞게 튜닝됩니다. 학습 데이터는 Mac 로컬에만 저장됩니다.

비파괴 보정 + XMP

밝기·컨트라스트·컬러·프리셋·복사 붙여넣기. 보정값은 XMP 사이드카에 저장되어 Lightroom에서 그대로 읽습니다. 양방향이라 Lightroom에서 만든 XMP도 PickShot이 인식합니다.

LOG 영상 자동 LUT

S-Log3 · V-Log · D-Log 메타데이터 자동 감지. 내가 설정한 LUT를 다음 클립부터 자동 적용. 프리미어 켜기 전에 영상 셀렉이 끝납니다.

자연어 의미 검색 (v8.9+)

"노을에서 웃는 여자" 같은 문장으로 검색. MobileCLIP-BLT 로컬 임베딩. 얼굴·옷·포즈·장면 다중 조합 가능. 전부 온디바이스 처리.

10

문제 해결

Q. 앱이 "손상된 파일" 경고로 열리지 않습니다
시스템 설정 → 개인정보 및 보안 → "확인 없이 열기"를 클릭하세요. 다운로드 직후 macOS가 한 번 차단하는 것이 정상입니다. 그래도 안 되면 터미널에서 xattr -cr /Applications/PickShot.app.
Q. 외장 SSD에서 썸네일이 느립니다
적극 캐시 모드(⚡)를 켜시거나, 허브를 통하지 말고 Mac에 직결하세요. USB 3.2 Gen 2 이상 케이블인지도 확인.
Q. 메모리카드 백업이 시작되지 않습니다
시스템 설정 → 개인정보 → 파일 및 폴더에서 PickShot의 이동식 볼륨 권한이 켜져 있는지 확인. 카드가 FAT32 또는 exFAT이어야 합니다 (ExFAT 권장).
Q. Lightroom에서 별점이 안 보입니다
PickShot 설정 → 내보내기 → XMP 사이드카 쓰기가 켜져 있어야 합니다. Lightroom 카탈로그에 추가한 뒤 메타데이터 → 파일에서 메타데이터 읽기 실행. G 픽은 Lightroom 컬러 레이블(녹색)로 매핑됩니다.
Q. 클라이언트 뷰어 링크가 열리지 않는다고 합니다
Google Drive 공유 설정에서 "링크를 가진 모든 사용자"가 선택돼 있는지 확인. 회사 계정은 정책상 외부 공유가 차단될 수 있습니다. 그 경우 개인 Google 계정으로 재연결하세요.
Q. 저장된 별점·G가 사라졌습니다
PickShot 메타데이터는 파일별 XMP + 로컬 SQLite 양쪽에 저장됩니다. 파일을 다른 폴더로 옮기셨다면 XMP가 같이 따라갔는지 확인. 그래도 안 보이면 설정 → 데이터베이스 → 폴더 다시 스캔.
Q. 크래시 로그를 어떻게 보내나요?
Help → 진단 리포트 보내기. 익명으로 스택 트레이스만 전송됩니다. 또는 GitHub Issues 에 직접 올려주셔도 됩니다.

자주 묻는 질문

Lightroom을 완전히 대체하나요?
아니요. PickShot은 셀렉·전달에 특화된 도구입니다. 심화 보정·프린트 컬러 매니지먼트는 Lightroom이 강합니다. "Lightroom 켜기 전에 쓰는 도구"로 생각하시는 게 정확합니다.
RAW 원본이 서버로 올라가나요?
절대 올라가지 않습니다. 모든 AI(얼굴·CLIP·장면)는 온디바이스. 클라이언트 뷰어는 2560px JPEG만 Google Drive에 올립니다.
Intel Mac에서도 쓸 만 한가요?
네. 다만 체감 속도가 Apple Silicon의 절반 정도입니다 (2,000장 RAW 기준 M1 1.5초, Intel 2.8초). 얼굴·CLIP 기능은 첫 스캔이 조금 길어질 수 있습니다.
카드 취소는 어떻게 하나요?
설정 → 계정 → 구독 관리에서 언제든 즉시 취소. 결제 주기 끝까지는 계속 사용 가능. 환불도 지원합니다.
팀 라이선스 있나요?
준비 중입니다. 현재는 개인 라이선스로 운영되며, 스튜디오·에이전시 문의는 kimjjang869@gmail.com 으로 주시면 개별 견적 가능합니다.
업데이트는 자동인가요?
네. 앱 시작 시 GitHub Releases 기준으로 최신 버전을 확인하고, 새 버전이 있으면 알림. 수동 업데이트를 원하시면 설정에서 비활성화 가능합니다.
