# PickShot 클라이언트 셀렉 — 기능 설계서

## 1. 문제 정의

### 현재 워크플로우 (고통스러움)

```
포토그래퍼                                 클라이언트 (신랑/신부)
━━━━━━━━                                 ━━━━━━━━━━━━━━━━━
1. RAW 원본 백업
2. 전체 사진 리사이즈 (시간 소요)
3. 웹하드/클라우드 업로드 (시간 소요)
                                          4. 다운로드 (용량 큼, 시간 소요)
                                          5. 뷰어 프로그램 없음 → 한장씩 열어봄
                                          6. 파일명 메모장에 적기 (햇갈림, 실수)
                                          7. 수정 요청 → 글로만 설명 (전달 안됨)
                                             "얼굴 슬림하게", "밝기 올려주세요"
                                             어느 부분인지 정확히 전달 불가
                                          8. 메신저/메일로 파일명 목록 전송
9. 파일명 일일이 대조 (실수 가능)
10. 수정 코멘트 일일이 대조 (놓침)
11. 왕복 수정 → 또 대조...
```

**문제점:**
- 왕복 3~5일 소요
- 파일명 대조 실수
- 수정 요청 전달 부정확 (글만으로 한계)
- 클라이언트가 별도 프로그램 설치해야 함
- 양쪽 다 스트레스

## 2. PickShot 해결책

### 핵심 원칙
1. **클라이언트는 앱 설치 없이 웹 브라우저만으로** (QR/링크)
2. **PC/모바일 동일한 UI** (반응형 웹)
3. **코멘트 + 펜 마크** (직관적 수정 요청)
4. **셀렉 결과 자동 연동** (.pickshot 파일)
5. **Google Drive 기반** (양방향 업로드 + 대용량)

### 새로운 워크플로우

```
포토그래퍼                                 클라이언트
━━━━━━━━                                 ━━━━━━━━
1. PickShot에서 폴더 열기
2. "클라이언트 셀렉 시작" 클릭
   → 자동 리사이즈 (1200px JPG)
   → Google Drive 업로드
   → QR/링크 생성
3. QR/링크 카톡으로 전달 (1분)
                                          4. QR 스캔 또는 링크 클릭
                                             (앱 설치 없음 — 웹 브라우저)
                                          5. 썸네일 + 미리보기로 편하게 감상
                                             SP 셀렉 (스페이스바/탭)
                                             방향키로 이동
                                          6. 수정 요청: 코멘트 + 펜으로 표시
                                             "얼굴 슬림하게" + 빨간 원 그리기
                                          7. "셀렉 완료" 클릭 → 자동 전송
                                             (.pickshot 파일 Google Drive에 저장)
8. PickShot에서 .pickshot 파일 열기
   → 원본 폴더와 자동 매칭
   → 셀렉된 사진 SP 표시
   → 코멘트 + 펜 마크 오버레이 표시
   → 바로 보정/내보내기 시작
```

**결과: 당일 완료, 실수 0, 직관적**

## 3. 왜 Google Drive인가?

1. **양방향 업로드** — 포토그래퍼도 업로드, 클라이언트도 .pickshot 파일 업로드 가능
2. **대용량** — 15GB 무료, 유료 플랜 2TB+
3. **권한 관리** —
   - 퍼블릭 링크 (간편, 누구나 접근)
   - 이메일 제한 (클라이언트 Google 계정만 접근 — 보안)
4. **API 잘 되어 있음** — OAuth + 파일 업로드/다운로드/공유 링크 생성
5. **클라이언트가 이미 사용** — 대부분 Google 계정 있음

### 권한 설정 옵션
- **퍼블릭**: 링크 있으면 누구나 접근 (간편)
- **이메일 제한**: 클라이언트 Gmail 입력 → 해당 계정만 접근 (보안)

## 4. 구현 구조

### 4-1. PickShot (macOS 앱) 측

#### 클라이언트 셀렉 시작
1. 사진 리사이즈 (1200px JPG, 백그라운드)
2. Google Drive에 폴더 생성 (세션 이름)
3. 리사이즈된 JPG 업로드 (진행률 표시)
4. .pickshot 메타데이터 JSON 생성 → 업로드
5. QR코드 + 공유 링크 생성
6. 권한 설정 (퍼블릭 or 이메일 제한)

#### 셀렉 결과 가져오기
1. Google Drive에서 .pickshot 파일 다운로드 (또는 직접 받기)
2. 원본 폴더 선택
3. 파일명 기반 자동 매칭
4. SP 셀렉 자동 적용
5. 코멘트 + 펜 마크 오버레이 표시

### 4-2. 웹 뷰어 (pickshot-viewer) 측

#### 기술 스택
- HTML/CSS/JS (프레임워크 없음 — 가벼움)
- 반응형 (PC/모바일 동일)
- Google Drive API (이미지 로드)
- Canvas API (펜 도구)

#### UI 구조

**PC (가로 레이아웃)**
```
┌──────────┬────────────────────────────┐
│ 썸네일    │                            │
│ (스크롤)  │       미리보기 (큰 이미지)   │
│          │                            │
│ [SP] ★   │                            │
│ [  ] ★   │                            │
│ [SP] ★   │  ┌──────────────────────┐  │
│ [  ] ★   │  │ 💬 코멘트 입력         │  │
│ [  ] ★   │  │ ✏️ 펜 도구 (빨/파/녹)  │  │
│ [SP] ★   │  │ ↩️ 되돌리기            │  │
│          │  └──────────────────────┘  │
├──────────┴────────────────────────────┤
│  [← 이전] [SP 셀렉] [다음 →]  [셀렉 완료]│
└───────────────────────────────────────┘
```

**모바일 (세로 레이아웃)**
```
┌─────────────────┐
│   미리보기       │
│   (스와이프)     │
│                 │
├─────────────────┤
│ [SP] [코멘트] [펜]│
├─────────────────┤
│ 썸네일 (가로 스크롤) │
│ ○ ○ ● ○ ○ ○ ○  │
├─────────────────┤
│ 💬 코멘트 입력    │
│ [셀렉 완료]      │
└─────────────────┘
```

#### 핵심 기능
1. **썸네일 탐색** — 클릭/방향키로 이동
2. **미리보기** — 확대/축소 (핀치/더블클릭)
3. **SP 셀렉** — 스페이스바 또는 탭으로 토글
4. **코멘트** — 텍스트 입력 (사진별)
5. **펜 도구** — Canvas 위에 그리기 (빨/파/녹, 원/화살표/자유선)
6. **셀렉 완료** — .pickshot 파일 생성 → Google Drive 저장 또는 다운로드

## 6. 전달 방식

### 방법 1: Google Drive 자동 저장 (권장)
1. 클라이언트가 셀렉 완료
2. .pickshot 파일이 같은 Google Drive 폴더에 자동 저장
3. 포토그래퍼가 PickShot에서 "셀렉 결과 가져오기" → 자동 다운로드

### 방법 2: 직접 전송
1. 클라이언트가 셀렉 완료
2. .pickshot 파일 다운로드
3. 카톡/메일로 포토그래퍼에게 전송
4. 포토그래퍼가 PickShot에서 .pickshot 파일 열기
